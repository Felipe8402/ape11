Atividade 3

Assim como não tive problemas até agora na utilização do Git, utilizei pouco o Gitflow para fazer alguns testes e entender o funcionamento do projeto. Nesse tempo, ficou claro que é específico para desenvolvimento de software entre equipes, e realmente, parece ser extremamente objetivo e funcional, sendo que a ideia, simplifica, é trabalhar com tipos de branchs, sendo que cada uma tem uma função específica. Abaixo algumas que retirei do projeto do Gitflow no Github.

Branch develop

É a branch que contém código em nível preparatório para o próximo deploy. Ou seja, quando features são terminadas, elas são juntadas com a branch develop, testadas (em conjunto, no caso de mais de uma feature), e somente depois as atualizações da branch develop passam por mais um processo para então ser juntadas com a branch master.

Branches feature/*
São branches no qual são desenvolvidos recursos novos para o projeto em questão. Essas branches tem por convenção nome começando com feature/ (exemplo: feature/new-layout) e são criadas a partir da branch develop (pois um recurso pode depender diretamente de outro recurso em algumas situações), e, ao final, são juntadas com a branch develop

Coloco abaixo a forma como a Atlassian (desenvolvedora do Trello e outras aplicações), definiu em seu site, o fluxo padrão do Gitflow.

O fluxo geral do Gitflow é:
• Uma ramificação de desenvolvimento é criada a partir da branch principal
• Uma ramificação de lançamento é criada a partir da ramificação de desenvolvimento
• Ramificações de recurso são criadas a partir da ramificação de desenvolvimento
• Quando um recurso é concluído, ele é mesclado na ramificação de desenvolvimento
• Quando a ramificação de lançamento é concluída, ela é mesclada nas ramificações de desenvolvimento e principal
• Caso um problema seja detectado na branch principal,uma ramificação de hotfix é criada a partir da principal
• Após a conclusão da ramificação de hotfix, ela é mesclada para as ramificações de desenvolvimento e principal

Agora coloco abaixo alguns comandos usados para fazer a configuração incial do Gitflow (fonte).

Comando Gitflow Função
git flow init Faz a inicialização do Gitflow em um repositório Git
git flow feature start MYFEATURE Esse comando cria um novo branch da funcionalidade baseado no 'develop' e alterna para ele
git flow feature finish MYFEATURE Mescla MYFEATURE no 'develop, remove o branch da funcionalidade e volta para o branch 'develop'
git flow feature publish MYFEATURE Publica uma funcionalidade no servidor remoto
git flow feature pull MYFEATURE Recebe uma funcionalidade publica e acompanha as alterações remota
